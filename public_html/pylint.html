Tutorial for Using Pylint

WHAT IS PYLINT?
Pylint is a programming tool that checks for errors in your Python code. It enforces Python coding standards and looks for certain types of error. Pylint also can suggest how blocks may be refactored and offer feedback about the code's complexity.
HOW DOES PYLINT WORK
Pylint analyzes your code and displays messages about it. It also displays statistics about the number of warnings and errors found in your Python files. These messages are classified under categories such as errors and warnings. Finally, the code is given an overall score, based on the number and severity of the warnings and errors.
WHY USE PYLINT?
In addition to helping experienced programmers track down coding errors, beginners can learn coding syntax and become a more aware programmer. Python uses recommended programming styles, which is captured in PEP 8, the "Style Guide for Python Code". Pylint can be a quick and easy way of seeing if your code adheres to PEP 8 and easily recognizable to other potential users.
HOW DO YOU START?
Install Python with
$ pip install pylint
Choose a program to examine. Try the command:
$ pylint [myfile].py
You can get a list of helpful features with
$ pylint --long-help
WHAT DOES THE EVALUATION OUTPUT LOOK LIKE?
“Your code has been rated at 10.00/10 (previous run: 10.00/10, +0.00)”
WHAT KINDS OF MESSAGES WILL PYLINT SHOW?
  • (C) Convention - for programming standard violation
  • (R) Refactor - for bad code smell
  • (W) Warning - for python specific problems
  • (E) Error - for probable bugs in the code
  • (F) Fatal - for an error occurred which prevented pylint from doing further processing
HOW CAN YOU CONFIGURE PYLINT?
Pylint uses several checkers. You can disable specific checkers or some of its messages or categories by specifying --disable=[symbol]. If you want to enable only some checkers or some message symbols, use --disable=all then --enable=[symbol] with [symbol] being a comma-separated list of checker names and message symbols.
CAN YOU MAKE A CONFIGURATION FILE?
You can specify preferred options for your setup with a configuration file to specify default values. You can specify this configuration file on the command line using the --rcfile option. Or Pylint can use a (hidden) configuration file called: .pylintrc
WHAT DO YOU DO NOW?
Start working through the Pylint errors! It will display syntax, linting, and other errors. Work through them one by one, repeating pylint as you go. Work until all errors are corrected, or if it highlights something that you don’t want to change, just disable that error.
